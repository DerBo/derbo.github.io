 数组(Array),就是相同数据类型的元素按一定顺序排列的集合,就是把有限个类型相同的变量用一个名字命名,
 以便统一管理它们,然后用编号区分他们,这个名字称为数组名,编号称为下标或索引(index),组成数组的各个
 变量称为数组的元素(element),数组中元素的个数称为数组的长度(length)。

一 、一维数组的定义方式

  • 1.静态初始化

    • 格式一 ：数据类型[] 数组名={元素1，元素2，元素3…}; //只能 在一个语句中完成，不能分开两个语句写

               比如：定义一个由几个整型组成的数组

         int[] arr = {1,2,3,4,5};//正确

         int[] arr;
         arr = {1,2,3,4,5};//错误

    • 格式二 ：数据类型[] 数组名 = new 数据类型 []{元素1，元素2，元素3…};

               比如：定义一个由几个整型组成的数组

         int[] arr = new int[]{1,2,3,4,5};//正确  

         int[] arr;
         arr = new int[]{1,2,3,4,5};//正确

  • 2.动态初始化

    • 格式三 ：数组存储的元素的数据类型[] 数组名字 = new 数组存储元素的数据类型[长度]； 或 数组存储的数据类型[]数组名字；

               数组名字 = new 数组存储的数据类型[长度]；

      注意：

      1. 数组的长度一旦指定后就不能更改，在容器和Arrays工具类中的动态数组，copy工具都是在内部新增 了一个更大的数组，然后将原数组的引用指向新数组。
      2. 数据局部变量的动态初始化是指new新数组并指定长度的时候，数组的元素会初始化为0，而不是nullSystem.out.println()输出的时候可以输出内容。而其他局部变量则必须初始化不然输出的时候会报错。

  • 3.数组的默认值

       不同的数组初始化的时候都会有它的默认值：

        数组元素类型       元素默认初始值

         byte                   0

         short                  0

         int                    0

         long                   0L

         float                  0.0F

         double                 0.0

         char                   0或为：'\u0000'

         boolean                false

         引用类型                null

二 、 数组在内存中的存储方式

       例如：

​            public static void main(String[] args) {

  	                int[] arr = new int[3];
                 	System.out.println(arr);
                } 

        ①arr是存放在方法栈中，记录的是数组的引用地址。

        ②new int[3]; 是在堆中创建的有三个连续空间的内存地址，它的首地址指向在栈中的arr。

三 、 数组使用过程中发生的异常

  • 一）数组越界异常（运行时异常）

        例如：

        public static void main(String[] args) {
                int[] arr = {1,2,3};
                System.out.println(arr[3]);//超出下标范围
        }
    • 此时数组的下标分别为距离首地址 0，1，2 长度的内存空间，当操作要执行arr[3],即下标为3的元素进行访问时，此时的访问下标已经超出了数组可以访问的下标就会抛出ArrayIndexOutOfBoundsException 异常，Java中的数组和c语言中的数组不同的地方在于，c语言中没有对用户输入的下标进行校验，所以如果程序员不谨慎的进行访问的情况下，编译器是不会警告这个问题的，所以有时会访问到非法的内存区域，而这在Java中不同，Java对每一个输入的下标都额外的进行了校验，虽然会牺牲一些维护消耗和内存，但显然这是值得的。

  • 二）数组空指针异常（运行时异常）

        例如：              
        public static void main(String[] args) {
                int[] arr = {1,2,3};
                arr = null;
                System.out.println(arr[0]);
        ｝
    • arr数组本来初始化为长度为3的数组，又赋值为1，2，3 而arr=null，则意味着变量arr将不会再保存数组的内存地址，也就不允许再操作数组了因此运行的时候就会抛出：NullPointerException 空指针异常。
